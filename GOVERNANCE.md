# Project Governance

This document describes the governance structure and decision-making model of AnomotionJS.

## Project Leadership
AnomotionJS is currently operated under a Technical Steering Committee (TSC) model:
* **Core Maintainers:** Lead design decisions, review pulls, and manage packages.
* **Contributors:** Review features, report bugs, and write documentation.

## Decision Making Process
Decisions are made using a **Lazy Consensus** model:
* A proposal is made via a GitHub Issue or Pull Request.
* Core maintainers have a 72-hour review period.
* If no objections (vetoes) are raised, the proposal is considered approved.

## Financial Transparency
All financial contributions are hosted on OpenCollective and are spent exclusively on hosting domains, test servers, and compensating major feature implementations.
